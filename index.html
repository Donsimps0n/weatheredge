Architecture

How WeatherEdge Actually Works

A Railway-hosted Flask API connects to Polymarket's Gamma API, fetches consensus forecasts from 4 weather models, calculates probabilistic edge with Bayesian statistics, and surfaces trade signals to the dashboard.

Scan→Forecast→Edge→Trade
STEP 01

Market Discovery + Forecasting

Queries Polymarket Gamma API for all weather-tagged events. Parses market titles to extract city, date, and temperature unit. Fetches GFS, ECMWF, UKMO, and NWS forecasts in parallel via Open-Meteo and api.weather.gov. Loads NOAA NCEI historical base rates (10 years) per city/date.

STEP 02

Edge Calculation + AI Analysis

Builds weighted consensus temperature distribution. Calculates dynamic σ based on hours-to-resolution and model spread. Runs Normal CDF probabilities for each outcome bin. Claude AI receives all data and returns structured decision: BUY_YES, BUY_NO, or SKIP with confidence tier and suggested position size.

STEP 03

Risk-Managed Execution

If edge exceeds threshold (3–8% dynamic), the Trader module executes via Polymarket CLOB. FOK market order, fallback to GTC limit with 2% slippage cap. Positions tracked with 5-layer exit monitoring. Quarter-Kelly sizing never risks more than 15% of bankroll on a single trade.
